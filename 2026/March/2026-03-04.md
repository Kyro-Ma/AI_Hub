## 2. 美国最高法院一锤定音，AI 作品不受版权保护
2026年3月2日美国最高法院宣布拒绝审理一项关于人工智能生成作品是否能获得版权的案件。案件涉及一位名叫斯蒂芬泰勒的计算机科学家，他为一台名为DABUS的人工智能系统创作了一幅名为《通往天堂的最新入口》的画作。他申请版权但被美国版权局拒绝，理由是版权保护需要人类作者参与。下级法院也支持了这一决定。泰勒的律师认为此案对未来的创意产业非常重要，尤其是在人工智能技术快速发展的背景下。他们指出，这幅画是人工智能独立完成的，不是人类参与创作。尽管其他艺术家曾为Midjourney等工具生成的图像申请版权，但他们的说法是人工智能只是辅助。此次最高法院不受理此案，意味着目前法律仍坚持只有人类才能成为版权作品的作者。

## 3. AI 解开罗马时代神秘石块谜团，推测出古代棋盘游戏规则
2026年3月2日，科学家在荷兰发现了一块罗马时代的白色石块，上面有斜线和直线，长期没人知道它的用途。研究人员用三维成像技术发现，有些刻痕更深，说明古代人曾用棋子在上面滑动。他们用一种叫卢迪的人工智能程序，学习了同一地区大约一百种古代游戏规则，让计算机自己尝试各种玩法，然后选出适合人类玩的版本。接着研究人员把这些规则和石块上的磨损痕迹对比，最终推测出一种游戏规则，目标是用最少的步数把对手的棋子追上并困住。虽然这个结果看起来合理，但专家提醒，人工智能可以模仿各种图案生成规则，不能完全证明古罗马人就是这么玩的。这项研究已经发表在《古物》期刊上。

## 10. Databricks推出面向AI工作负载的PostgreSQL数据库Lakebase
Databricks公司推出了一个名为Lakebase的新数据库服务，它基于PostgreSQL打造，可以自动调整计算和存储资源，适合处理实时数据和人工智能任务。这个服务能让用户在同一个平台上同时进行数据操作和数据分析，避免了传统数据库因为资源共享而容易出问题的情况。Lakebase支持创建数据分支、随时恢复到过去某个时间点，还能快速生成数据副本，让开发更高效。它能直接与Databricks的分析工具连接，无需额外转换数据格式。服务目前在亚马逊云平台上可以用于生产环境，在Azure上处于公开测试阶段，未来几个月将全面支持Azure和谷歌云。Lakebase提供两种版本，一种是自动调整资源的版本，另一种是预先设置好的版本。使用费用根据实际使用情况计算，存储费用单独计费。公司计划在2026年初获得安全认证。

## 11. Rivet 发布 Sandbox Agent SDK，解决 Agent API 碎片化问题
Rivet公司推出了一款名为Sandbox Agent SDK的工具，帮助开发者更容易地搭建能自主工作的AI代理。过去不同AI服务比如Claude Code、Codex、OpenCode和Amp都有各自不同的接口，开发者需要为每个服务单独编写代码，非常麻烦。这款新工具让开发者只需写一套通用代码，通过修改设置就能切换使用不同的AI服务。它统一了会话管理、数据格式和错误处理，让程序运行更稳定，输出信息也更容易读取。所有事件都按照相同规则传递，开发者无需再为不同服务分别处理。该工具用Rust语言编写，体积小，运行简单，可以在多种环境如Docker或云沙箱中使用。它就像一个桥梁，把原本零散的AI服务连接起来，让开发变得更简单高效。

## 13. 从零构建低于 500ms 的语音代理：端点检测、架构权衡与工程优化
有人分享了自己搭建语音助手的过程，整个过程从说话到听到回复的时间低于半秒。这个系统通过语音转文字，再让电脑理解内容，最后把回答变成声音来完成。为了更自然，系统会判断用户什么时候说完话，避免提前或延迟打断。有人用Deepgram的Flux模型来判断说话结束，效果比传统方法更好。系统还通过提前准备声音、把计算放在靠近用户的地方、使用缓存等方式减少延迟。虽然直接从语音生成语音的模型理论上能更快，但需要大量计算资源，而且难以保证安全和可查。企业更喜欢分步骤处理，这样更容易检查和控制。实际使用中，用户对延迟的容忍度不同，有些会感觉太慢，有些则觉得差不多。大公司因为成本高、安全要求严格，没有快速推广这类产品。
