## 1. 海淀放大招！90亿资金+近30条政策，重磅释放三大科创关键信号

## 2. Anthropic 遭“封杀”后，OpenAI 与美国国防部达成协议
2026年3月1日，OPENAI宣布与美国国防部达成协议，允许其人工智能模型在机密网络中使用。此前，美国国防部与竞争对手Anthropic发生争执，指责后者拒绝让技术用于军事用途，甚至可能被用于大规模监控和自主武器。Anthropic方面表示，他们并不反对军事行动，但担心人工智能会损害民主价值观。超过六十名OPENAI员工和三百名谷歌员工联名呼吁支持Anthropic的立场。美国政府随后下令在六个月后停止使用Anthropic的产品，并将其列为供应链风险。OPENAI则强调，其协议中明确禁止国内大规模监控，所有武力使用必须由人类控制，且模型若拒绝执行任务，政府不会强迫其运行。OPENAI还承诺派驻工程师协助安全部署，希望政府能以公平方式对待所有人工智能公司。

## 3. 谷歌 Nano Banana 2 一夜补齐短板，各种图解都能画，价格才是 OpenAI 一半
谷歌推出的Nano Banana 2在文生图领域取得重大突破，它不仅能生成清晰逼真的图片，还能实时联网查询信息，让画面更贴近真实世界。比如生成街景时，广告牌、路牌等细节都准确无误，甚至能根据地理位置和天气变化调整画面。它还能把复杂知识变成清晰图表，如解释水循环或制作食谱图，帮助学生和研究人员更好理解内容。图片中的文字也更准确，支持多种语言翻译，风格不变。人物和物体在不同场景中保持一致，换颜色或材质也容易。图片分辨率从512到4K可选，适合不同用途。整体画质更自然，细节更丰富，使用体验更流畅。这款模型价格只有OPENAI的一半，性价比很高，已在谷歌多个产品中上线，成为当前最受欢迎的图像生成工具之一。

## 4. Werld：用 NEAT 进化拓扑构建的代理文明，涌现行为与叙事流
Werld 是一个用电脑模拟的代理文明游戏，里面有很多小生命体生活在由节点和连线组成的世界上，它们的思维结构会随着生存情况慢慢变化。这个项目使用一种叫 NEAT 的方法让这些小生命体的大脑结构不断进化，不需要人为设定具体行为。世界中的能量、地形和季节等规则被当作自然规律，影响生命体的生存和繁殖。系统没有明确的奖励机制，而是通过谁活得久、能生孩子来决定谁更“成功”。评论者关注这些规则是否隐藏了人为偏见，以及生命体会不会出现类似自我刺激的行为。项目还尝试用大型语言模型生成故事，让整个过程像小说一样展开。有人提出文档不够清楚，界面和说明需要改进，同时运行时也遇到一些问题，比如生命体无法存活或繁殖失败。整体来看，这是一个探索生命如何自然出现的有趣尝试。

## 5. 研究证实：AI 大模型跨多轮对话任务表现不佳，性能最高降 39%
2026年的一项研究发现，目前的大语言模型在需要多次对话才能完成的任务中表现不好，当信息被分成多条消息发送时，模型的准确率会明显下降，性能最高可能降低百分之三十九。虽然新版本模型比旧版本稍好一些，降幅从三十九个百分点减少到三十三个百分点，但问题仍然存在。在一些任务中比如写程序，模型的损失只有百分之十到百分之二十。研究人员指出，如果用户在对话过程中改变主意，实际表现可能更差。调整模型的某些参数并不能解决问题。专家建议，如果对话出错，应该重新开始一段新的对话，先让模型把所有问题整理清楚，再从整理好的内容开始继续对话。这项研究测试了代码、数据库、操作指令等六个方面，结果表明模型在真实使用场景中的表现可能比测试中更差。

## 6. 消息称 Meta 自研芯片梦受挫：最先进芯片项目夭折，靠合作寻求替代方案
2026年2月消息显示，Meta公司放弃了自己研发最先进AI训练芯片的计划，这个项目原本希望打造能和英伟达媲美的芯片，但因为设计太复杂、耗电量大、开发周期长，最终决定停止。公司现在转而选择和外部供应商合作，比如和谷歌、AMD以及英伟达建立合作关系，使用他们的AI芯片来支持自己的数据中心。Meta之前尝试过两种芯片设计，一种是结构简单但软件难用，另一种是更先进但技术难度高，最终都未能成功。公司认为现有供应商的芯片软件更成熟，使用起来更安全可靠，能更快地训练新模型，因此决定暂时不继续自己研发，而是依靠合作来推进AI发展。

## 8. 对话冯雷：从 AI 播客，到真人感 Agent
冯雷是ListenHub的创始人，他选择做AI播客这个方向，因为很多人虽然喜欢短视频，但仍然愿意听播客，尤其是那些有深度内容的。他的公司不追求用户数量，而是专注于服务一小部分有创造力、愿意花钱的创作者。产品强调自然真实，像真人说话一样，不生硬。公司用海外的AI模型做内容生成，中文处理也用国内模型，但英文和多语言能力仍需提升。他们不靠免费吸引用户，而是通过工具收费，去年收入达到三十万美金，估值达到五千万美金。公司坚持用真实数据和用户反馈来调整产品，不盲目照搬旧方法。他们正在开发一个能像真人一样思考和行动的智能助手，未来会推出这款产品。

## 9. MicroGPT：约 1000 行 C 实现的可读 GPT-2，暴露 KV-cache 与内存布局
karpathy 发布了一个约二百行的简单程序 microGPT，用 Python 实现了类似 GPT 的模型，目的是帮助人们理解神经网络的基本原理。这个程序把注意力机制简化成容易看懂的代码，让人能亲手操作看到每个步骤。社区有人把它翻译成 C、Rust 或 C++，也有人尝试在浏览器里运行。大家讨论了这个程序在生成内容时如何保存之前的信息以提高速度，避免重复计算。也有评论质疑网络上出现大量错误说法，比如误称是千行 C 代码，这些内容可能来自机器生成。有人认为模型的核心思想其实不难，但真正达到高级智能还需要更多训练和真实世界经验。普通人用普通电脑也能尝试训练小模型，但需要时间和耐心。

## 10. Xmloxide：用 Claude Code 生成的 Rust 替代 libxml2，体积更小且公开 API 无 unsafe，引发维护与安全讨论
一位开发者用AI工具生成了一个新的Rust版本的XML解析库叫xmloxide，它比传统的libxml2更小，对外提供的接口没有使用危险的代码部分，这让使用者更安全。这个项目通过自动测试和检查工具不断改进，减少了人工干预。有人担心AI生成的代码可能有风险，建议在使用时加上提醒说明。虽然库的内部实现可能仍有一些危险操作，但对外接口承诺安全，让普通用户可以放心使用。相比libxml2，xmloxide的代码量大约是四万行，而原库有十五万行，说明它更简洁。评论中也提到，很多公司用libxml2却不参与维护，导致安全问题长期存在，有人呼吁政府或企业给予开源维护者支持。同时有观点认为AI工具可能帮助降低维护成本，但不能完全替代人工维护。

## 11. Woxi：用 Rust 重写 Mathematica 引发的架构、性能与 AI 贡献争议
Woxi 是一个用 Rust 语言重写 Mathematica 的开源项目，目标是实现超过九百个常用数学函数。项目在开发中大量使用人工智能工具生成代码，并通过数千个测试来验证功能。有人担心人工智能生成的代码难以检查，可能存在错误，也有人认为这能加快开发速度。项目团队提出将数学规则写在一种叫 woxilang 的语言里，而不是直接写在 Rust 中，这样更容易让数学专家参与改进。目前项目在算法深度和功能完整度上还不及 Mathematica，前端体验和数学排版也需进一步提升。项目采用 AGPL 许可证，这意味着在云端使用时必须公开代码，可能影响商业应用。能否真正替代 Mathematica 还有待观察。

## 12. Now I Get It：用 LLM 把论文变成交互式网页——演示、幻觉与配额问题
现在有一个项目叫Now I Get It，它能用人工智能把学术论文变成可以互动的网页，帮助人们更容易理解论文内容。作者已经测试了百篇论文，发现生成的网页有时会出现错误，比如解释错误或画出原本没有的图表，这被称为幻觉问题。为了提高质量，项目加入了重新生成和人工检查的功能。由于使用人工智能的成本较高，每天处理的论文数量有限。项目还面临技术难题，比如PDF文件很难直接读取，需要先转成图片再识别文字。用户希望网页能有更多功能，比如自动查找引用资料、支持图表重做和切换不同模型。有人建议通过付费或捐款来支持这个项目，也有人提出可以自己搭建类似服务。整个过程需要平衡成本和用户体验，未来可能通过更好的设计和功能吸引更多人使用。

## 13. Claude 登顶美区 App Store：性能、广告、下载潮与政治信任之争
Claude在美区App Store登顶引发网友热议，大家争论这是否代表其技术真正更强。有人认为Claude的对话速度和质量优于OPENAI的GPT系列，也有人指出这可能只是短期下载潮导致的排名变化。App Store的榜单主要看最近有多少人下载和使用，所以短时间内热度上升并不等于长期领先。还有人提到，公众对OPENAI的商业和政治关联存在疑虑，而Anthropic更强调安全和伦理，因此赢得了一些用户的信任。部分用户因为广告吸引或想尝试新工具，主动卸载了ChatGPT转而使用Claude。不过也有评论提醒，榜单上有时会出现与应用无关的项目，比如零售店，这说明排名不能完全反映产品好坏。真正决定谁更优秀，还需要长期观察各种功能表现。

## 14. Qwen3.5（122B/35B）宣称本地可达 Sonnet 4.5 水平，社区质疑基准优化、量化与硬件瓶颈
阿里巴巴发布了Qwen3.5系列模型，声称在本地设备上运行可以达到Anthropic公司Sonnet 4.5的水平。这一说法引发了社区广泛讨论，许多人质疑这种比较是否真实可靠。因为公开的测试题目容易被专门优化，模型在这些题目上表现好并不代表真实使用中的能力。本地运行受到硬件条件和软件工具的很大影响，比如显存大小、使用的量化方式和推理软件等，这些都会改变模型的实际表现。一些用户发现，使用合适的工具和参数后，Qwen3.5在编程和短文本任务中表现不错，但要达到高端模型的水平仍需更多条件。同时，部分人对使用中国开发的模型在敏感场景下是否安全表示担忧。总体来看，模型的实际表现取决于多种因素，不能简单地用一个分数来判断。

## 15. OpenAI 因员工在预测市场上内幕交易被解雇
OPENAI 因一名员工在外部预测市场使用公司内部信息进行交易而解雇。该员工通过多个加密钱包下注，交易时间点和账户创建模式被发现异常。调查人员通过分析资金流入时间、钱包关联和交易记录来判断是否涉及内幕交易。预测市场在美国由商品期货交易委员会监管，其对内幕交易的认定标准与股票市场不同，执法力度较弱。不同平台规则也各不相同，有的平台明确禁止内幕交易，有的则更宽松。有人认为内幕信息有助于提高预测准确性，但也担心会引发市场操纵或影响公共决策。员工使用加密钱包时可能通过创建多个账户或拆分资金来隐藏行为，这增加了调查难度。公司强调员工不得利用机密信息获利，解雇是为维护公司信誉和保密制度。目前尚无刑事起诉，公众对事件的性质和处理方式仍有疑问。
